Base

  • git config --global user.name blabla - встановлює ім'я користувача
  • git config --global user.email blabal@bla - встановлює пошту користувача
  • git init - створити новий репозиторій
  • git status - показати статус файлів в робочому репозиторії
  • git add - додати вибрані файли в staging area, які потім будуть закомічені
  • git add . - додати всі файли каталогу до staging area
  • git commit - записати зміни до репозиторію
  • git commit -m "опис коміту" - створити коміт з описом
  • git log - показати інформація про попередні коміти
  • git checkout - переміститися на коміт або гілку, також оновлює робочу директорію до стану вибраної гілки
  • git branch - створити нову гілку
  • вигрузити зміни на сервер
  • отримати зміни з серверу
  • git merge - злиття гілок в одну
  • git clone - скачує репозиторій з усією історією змін

Написати що таке.gitignore файл

Це файли, які не відслідковуються. Ці файли не будуть комітитись і не будуть показуватись в git status

Advanced

  • git revert - скасувати коміт
  • git reset - видалити всі коміти вище вказаного
  • git reset --hard [коммит] - видаляє історію разом зі станом робочої директорії до вказаного коміта
  • git rebase - дозволяє перетаскувати коміти з однієї гілки в іншу
  • git cherry-pick - стягнути зміни з певного комміту і примінити їх до нового коміту, без лиття гілок
  • git stash - тимчасово ховає зміни у файлі, щоб можна було продовжити роботу з попередньою версією файлу, а потім знову примінити зміни

Bonus

  • ls - показати всі файли, які знаходяться в папці
  • cd <шлях папки> - перейти в папку
  • ssh-keygen - генерує SSH ключ
  • # start the ssh-agent in the background - відкрити ssh-агент
  • ssh-add ~/.ssh/id_rsa - додати ssh-ключ
